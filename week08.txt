# 1. ¿Qué es CSS Flexbox?

**Flexbox** (Flexible Box Layout) es una herramienta de CSS que ayuda a organizar elementos dentro de un contenedor de forma flexible y ordenada.

Piensa en Flexbox como una manera fácil de alinear cajas en una fila o una columna sin tener que usar trucos complicados.

### Ejemplo visual

Sin Flexbox:

```
[Botón 1]
[Botón 2]
[Botón 3]
```

Con Flexbox:

```
[Botón 1] [Botón 2] [Botón 3]
```

O incluso:

```
      [Botón 1]
      [Botón 2]
      [Botón 3]
```

Todo controlado con unas pocas líneas de CSS.

---

# 2. ¿Cómo puedo usar Flexbox para diseñar páginas web?

Primero se convierte un contenedor en un contenedor flexible:

```css
.contenedor {
  display: flex;
}
```

HTML:

```html
<div class="contenedor">
  <div>Elemento 1</div>
  <div>Elemento 2</div>
  <div>Elemento 3</div>
</div>
```

### Propiedades más comunes

### Colocar elementos en fila

```css
display: flex;
flex-direction: row;
```

Resultado:

```
[1] [2] [3]
```

---

### Colocar elementos en columna

```css
display: flex;
flex-direction: column;
```

Resultado:

```
[1]
[2]
[3]
```

---

### Centrar elementos

```css
display: flex;
justify-content: center;
align-items: center;
```

Esto centra el contenido horizontal y verticalmente.

---

### Separar elementos uniformemente

```css
display: flex;
justify-content: space-between;
```

Resultado:

```
[1]          [2]          [3]
```

# 3. ¿Cuál es la diferencia entre CSS Grid y Flexbox?

Ambos sirven para organizar elementos, pero tienen propósitos distintos.

| Flexbox                         | Grid                             |
| ------------------------------- | -------------------------------- |
| Trabaja en una dimensión        | Trabaja en dos dimensiones       |
| Filas o columnas                | Filas y columnas al mismo tiempo |
| Ideal para componentes pequeños | Ideal para diseños completos     |
| Más simple                      | Más potente                      |

### Flexbox

Piensa en una sola línea:

```
[1] [2] [3] [4]
```

---

### Grid

Piensa en una cuadrícula:

```
[1] [2] [3]

[4] [5] [6]

[7] [8] [9]
```

---

# 4. ¿Cuándo debería usar Flexbox en lugar de Grid?

### Usa Flexbox cuando:

✅ Quieras alinear botones.

✅ Crear menús de navegación.

✅ Organizar tarjetas en una fila.

✅ Centrar contenido.

✅ Trabajar en una sola dirección (horizontal o vertical).

Ejemplo:

```html
Inicio | Servicios | Contacto
```

Flexbox es perfecto.

---

### Usa Grid cuando:

✅ Diseñes toda la estructura de una página.

✅ Tengas varias filas y columnas.

✅ Necesites controlar áreas complejas.

Ejemplo:

```
+------------------+
| Header           |
+------------------+
| Sidebar | Main   |
+------------------+
| Footer           |
+------------------+
```

Grid es la mejor opción.

# 5. ¿Qué es la herramienta Chrome Lighthouse?

**Lighthouse** es una herramienta gratuita integrada en el navegador Chrome que analiza la calidad de una página web.

Genera un reporte con puntuaciones sobre:

* Rendimiento (Performance)
* Accesibilidad (Accessibility)
* Buenas prácticas (Best Practices)
* SEO
* Progressive Web Apps (PWA)

---

### Cómo abrir Lighthouse

1. Abre la página web.
2. Presiona **F12**.
3. Ve a la pestaña **Lighthouse**.
4. Haz clic en **Analyze page load**.

Chrome generará un informe.

---

# 6. ¿Cómo puede utilizarse para optimizar páginas web?

Lighthouse detecta problemas y sugiere mejoras.

### Ejemplo

Puede indicar:

❌ Imágenes demasiado grandes.

```
Imagen: 4 MB
```

Sugerencia:

```
Reducir a 300 KB
```

---

También puede detectar:

* JavaScript innecesario.
* CSS sin usar.
* Texto difícil de leer.
* Falta de etiquetas SEO.
* Problemas de accesibilidad.

---

### Beneficios

✔ Página más rápida.

✔ Mejor experiencia para usuarios.

✔ Mejor posicionamiento en Google.

✔ Menor tasa de abandono.

# 7. ¿Qué son las Meta Tags de redes sociales en HTML?

Son etiquetas dentro del `<head>` que indican a Facebook, LinkedIn, X (Twitter) y otras plataformas qué mostrar cuando alguien comparte tu página.

Sin estas etiquetas:

```
Título incorrecto
Imagen aleatoria
Descripción vacía
```

Con ellas:

```
Imagen atractiva
Título correcto
Descripción profesional
```

---

# 8. ¿Por qué son importantes?

Porque mejoran la apariencia de tus enlaces compartidos.

### Sin Meta Tags

```
misitio.com/articulo
```

Puede verse así:

```
Título desconocido
Sin imagen
```

---

### Con Meta Tags

```
📷 Imagen
Título del artículo
Descripción atractiva
```

Esto aumenta los clics y la interacción.

# 9. ¿Cuáles debería usar?

## Open Graph (Facebook, LinkedIn, WhatsApp)

```html
<meta property="og:title" content="Mi Sitio Web">
<meta property="og:description" content="Descripción de la página">
<meta property="og:image" content="https://misitio.com/imagen.jpg">
<meta property="og:url" content="https://misitio.com">
<meta property="og:type" content="website">
```

---

## Twitter Cards

```html
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="Mi Sitio Web">
<meta name="twitter:description" content="Descripción">
<meta name="twitter:image" content="https://misitio.com/imagen.jpg">
```

---

## Meta Tag SEO básica

```html
<meta name="description"
      content="Descripción para Google">
```

---

## Problemas que pueden surgir si no usas estas herramientas

### Sin Flexbox/Grid

* Diseños desordenados.
* Difícil adaptación a móviles.
* Más código CSS.

### Sin Lighthouse

* Sitio lento.
* Mala experiencia de usuario.
* Menor posicionamiento SEO.

### Sin Meta Tags sociales

* Enlaces compartidos sin imagen.
* Menos clics.
* Apariencia poco profesional.

---

## Ejemplo real

Imagina una inmobiliaria que comparte una propiedad en Facebook.

**Sin Open Graph:**

```
www.casas.com/propiedad123

Sin imagen
Título raro
Sin descripción
```

**Con Open Graph:**

```
📷 Foto de la casa

Casa en Miami por $450,000

3 habitaciones, piscina y garaje
```

La segunda opción genera muchos más clics y consultas.
